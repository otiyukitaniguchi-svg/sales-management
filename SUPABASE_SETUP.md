# Supabase スキーマ実行ガイド（段階的）

エラーを避けるため、4つのステップに分けて実行します。

---

## 📋 実行手順

### ステップ1: テーブル作成（必須）

1. Supabase → **SQL Editor** → **+ New query**
2. `supabase-schema-step1.sql` の内容を**全てコピー&ペースト**
3. **Run** をクリック
4. 「ステップ1完了: テーブル作成成功」と表示されればOK

**エラーが出た場合:**
- エラーメッセージをスクリーンショット
- 「テーブルが既に存在します」→ 問題なし、次へ進む

---

### ステップ2: インデックス作成（推奨）

1. **+ New query** で新しいクエリを作成
2. `supabase-schema-step2.sql` の内容を**全てコピー&ペースト**
3. **Run** をクリック
4. 「ステップ2完了: インデックス作成成功」と表示されればOK

**効果:** 検索が高速化されます

---

### ステップ3: トリガー作成（推奨）

1. **+ New query** で新しいクエリを作成

4. 「ステップ3完了: トリガー作成成功」と表示されればOK

**効果:** レコード更新時に自動でタイムスタンプが更新されます

---

### ステップ4: セキュリティ設定（オプション）

## 🆘 よくあるエラーと対処法

### エラー1: "extension "uuid-ossp" does not exist"

**対処法:**
```sql
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
```
だけを先に実行してから、ステップ1を再実行

### エラー2: "table already exists"

**対処法:** 問題なし。そのまま次のステップへ進む

### エラー3: "syntax error near ..."

**対処法:**
1. コピー時に余分な文字が入っていないか確認
2. 全選択してコピーし直す
3. それでもダメならエラーメッセージを教えてください
